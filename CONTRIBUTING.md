# Contributing guidelines

All efforts to contribute are welcome and encouraged, as the project can only be better with more eyes looking for problems and more minds trying to solve them.

All contributions must be inline with our code of conduct.

Ways to contribute:

## Reporting issues

Think you've found a problem? We'd like to hear about it!

How to report via the [issue tracker](https://github.com/PurpleKingdomGames/indigoengine/issues):

> Please raise ONE TICKET PER ISSUE.

1. Try and search for similar issues currently open.
2. If you can't find an issue, raise a new one.
3. Please try to describe a minimal scenario that reproduces the issue.

## Updating documentation

Game engines and their supporting tools are complicated, especially for new people, and so are only as good as their documentation. If you like to help update our docs, please raise a Pull Request against the [documentation website repo](https://github.com/PurpleKingdomGames/ultraviolet-docs).

## Donations and sponsorship

Contributions are based on time or money, and building and maintaining Ultraviolet requires both. If you'd like to help cover our costs, help pay for development or just buy us a cup of coffee, we'll be very grateful.

Please look for the sponsorship button on the [github repository](https://github.com/PurpleKingdomGames/indigoengine).
